算法三阶段：

  1. 用编程语言能够解决问题
  2. 熟悉算法背后的逻辑，可以使用纸笔解释清楚，证明其准确性与分析大O运行时
  3. 第一次提交前多做检查，尽可能保证准确

一、复杂度

以时间或空间衡量算法效率。

时间复杂度：算法执行操作的次数随输入规模 n 的增长趋势。计算：子问题个数 × 解决子问题时间。 空间复杂度：算法运行过程中额外占用的内存随 n 的增长趋势。若只使用常数级额外空间（O(1)），称为原地（in-place） 算法。

常见复杂度（从优到劣）：O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)

二、数据结构

常见数据结构可分为「线性」与「非线性」。

线性：元素之间一对一 树形：元素之间一对多 图形：元素之间多对多

存储方式

两种底层存储：数组（顺序存储） 和 链表（链式存储）。

  • 数组：紧凑连续，可随机访问，索引快，相对节约空间；一次性分配、扩容需整体复制、插入删除需移动元素。
  • 链表：元素不连续，依靠指针指向下一个；不能随机访问，消耗更多空间（指针）。

基本操作

遍历 + 访问（增删改查）。

  • 线性：for/while 迭代为代表
  • 非线性：递归为代表

常见数据结构

  • 栈：先进后出（LIFO）。队列：先进先出（FIFO）；双向队列两端可进出。栈实现队列用双栈；队列实现栈把队首取出放队尾。
  • 哈希表：映射，解决冲突：链式地址法、开放地址法。结合哈希表和数组可使删除和查找 O(1)（待操作元素与数组尾部交换）。
  • 链表：递归反转的关键 → 将子问题看作已完成的状态，补充 base case。反转整个 / 前 N 个 / 部分 [m,n] / 每 K 个一组。判断回文可反转后比较、入栈出栈比较、或借助后序遍历。
  • 树：遍历方式 → 层序 BFS；前、中、后序 DFS。递归算法：明确函数定义，清楚当前 root 做什么，再递归子节点。快速排序 = 前序遍历，归并排序 = 后序遍历。很多 DP 问题就是在遍历树；回溯是 N 叉树前后序遍历。
  • BST：节点值大于左子树、小于右子树；中序遍历为升序。基于 BST 有 AVL、红黑树、B+ 树等。操作：查找、插入、删除（末端直接删；一个子节点接替；两个子节点找左最大或右最小接替）。
  • 堆：大/小顶堆（完全二叉树）。二叉堆存储在数组中（parent: i/2，left: i*2，right: i*2+1）。insert 上浮（swim），delMax 下沉（sink）。
  • 图：顶点和边的集合；遍历 BFS、DFS。二分图：两种颜色着色，使任意边两端颜色不同。
  • 跳表：多层级链表，顶层是子层的子集。

三、算法范式

迭代与递归

  • 递归一定是自上而下的
  • 任何递归都可以通过循环加栈的迭代形式完成
  • 任何迭代形式的循环都可以通过递归形式完成
  • 递归适用于：类树形结构（子结构与父结构类似）、涉及回溯、深度不太大

分治法（Divide-and-Conquer）

将原问题划分成 n 个规模较小、结构相似的独立子问题，递归求解，再合并结果。

三个步骤：

  1. 分解（Divide）：将原问题分解为子问题
  2. 解决（Conquer）：递归求解子问题，足够小则直接求解
  3. 合并（Combine）：将子问题的结果合并为原问题的解

典型例子 — 归并排序：

  • 分解：n 个元素分成各含 n/2 个元素的子序列
  • 解决：递归排序两个子序列
  • 合并：合并两个已排序的子序列

动态规划（DP）

分治法 —— 各子问题独立； 动态规划 —— 各子问题重叠。

DP 问题的一般形式：求最值。核心思想：穷举求最值。

两个要素

  1. 最优子结构：问题的最优解包含子问题的最优解，可从子问题最优结果推导更大规模问题的最优结果。子问题之间互相独立。
  2. 重叠子问题：递归算法会反复求解同样的子问题。

三要素与框架

重叠子问题、最优子结构、状态转移方程。

思维框架：明确 base case → 明确状态 → 明确选择 → 定义 DP 数组/函数的含义。初始化 base case，遍历所有状态组合，对每个状态求最值（从可选的选择中）。

设计四步骤

  1. 描述最优解的结构
  2. 递归定义最优解的值
  3. 按自底向上的方式计算最优解的值
  4. 由计算出的结果构造一个最优解

两种实现

  • 自顶向下（记忆化搜索/备忘录）：递归拆解子问题，记录子问题的解
  • 自底向上（DP table 迭代）：循环迭代完成，脱离递归

状态压缩

二维 dp 数组，若计算 dp[i][j] 只需相邻状态，可投影到一维数组，将无法投影的值用临时变量存储。

DP 遍历

遍历过程中，所需状态必须已经计算出来；遍历终点是存储结果的位置。

经典 DP 问题

斐波那契数列

  1. 暴力递归 O(2^n)：存在大量重叠子问题
  2. 备忘录递归 O(n)：数组/哈希表缓存已计算结果
  3. DP table 迭代 O(n)：自底向上；状态压缩后只需两个变量

凑零钱（k 种面值，求凑出目标金额的最少硬币数）

  • 状态转移方程：base case（金额 0 → 0 枚）；状态（目标金额）；选择（每种硬币）
  1. 暴力递归 → 2. 备忘录 → 3. DP table 迭代（状态从 0 到目标）

股票买卖（最多 k 笔交易）

三个状态：天数 n、交易次数 K、持有状态（0/1）→ dp[n][k][0 or 1]。今天不持有 = max(昨天不持有, 昨天持有+今天卖出)；今天持有 = max(昨天持有, 昨天未持有-今天买入)。

打家劫舍

  1. 线性排列：抢或不抢，dp[i] 只与最近两个状态有关
  2. 环形排列：比较「抢首不抢尾」与「抢尾不抢首」
  3. 二叉树：每个节点返回 [不抢最大值, 抢最大值]，抢则加左右不抢的值，不抢则左右各取max。

0-1 背包 / 分割等和子集

  • 背包问题：N 个物品、重量 W，求最大价值
  • 分割等和子集：转化为背包 → 数组 sum/2 容量能否恰好装满
  • dp[i][j]：前 i 个物品、容量 j 时能否装满；可状态压缩为一维

贪心算法

每个决策阶段选择当前最优解，期望获取全局最优解。

与 DP 的区别：

  • DP 自底向上，每步选择依赖子问题的解
  • 贪心自顶向下，先做贪心选择，再求解剩余一个子问题
  • 贪心划分子问题后通常只剩一个非空子问题

适用条件：问题具有最优子结构，且贪心选择性质成立。

回溯法（Backtracking）

本质：DFS 暴力穷举，决策树的遍历过程。

三要素：1. 路径（已作出的选择）2. 选择列表（当前可作的选择）3. 结束条件。流程：满足结束条件则记录路径；否则遍历选择列表 → 做选择 → 递归 → 撤销选择。

回溯的路径、选择列表、结束条件 与 DP 的状态、选择、base case 对应；回溯自顶向下，DP 自底向上。

经典问题

  1. 全排列：DFS，选择节点进入下层，遍历后撤销。
  2. N 皇后：每行对应决策树每层，判断列冲突、左上/右上对角线冲突。

分支定界法（Branch and Bound）

与回溯法的主要区别：BFS 而非 DFS。

  • 每个活结点只有一次机会成为扩展结点，一次性产生所有儿子结点
  • 不可行或非最优的儿子结点被舍弃，其余加入活结点表
  • 从活结点表取下一节点继续扩展

选择下一节点的方法：

  1. FIFO：队列
  2. 最小耗费/最大收益：优先队列（最小堆/最大堆）

回溯法内存 O(解空间最长路径长度)，分支定界法内存 O(解空间大小)。 通常分支定界法先超出内存上限，回溯法先超出时间上限。

四、常用技巧

双指针

快慢指针（链表问题为主）

  1. 判断链表是否有环：一快一慢，有环则相遇
  2. 环的起始位置：第一次相遇后，一个回到头部，再次相遇即环起点
  3. 链表中点：快到尾部时，慢在中部
  4. 倒数第 K 个元素：快先走 K 步

左右指针（数组问题为主）

二分查找、两数之和、反转数组、滑动窗口。

原地修改数组（快慢指针）

  1. 删除排序数组重复项
  2. 删除有序列表重复项
  3. 移除数组中等于 val 的元素：fast 遇到 val 跳过，否则赋值给 slow
  4. 移动零到末尾：移除 0 后，后面全赋 0

二分搜索（Binary Search）

细节：mid 加一还是减一，while 用 <= 还是 <。mid = left + (right - left) / 2 防溢出。

搜索一个元素

  • 闭区间 [left, right]，right = len - 1，while(left <= right)
  • 终止条件 left == right + 1
  • left = mid + 1, right = mid - 1

搜索左边界

  • 左闭右开 [left, right)，right = len，while(left < right)
  • 找到 target 时 right = mid（收缩右边界）
  • 返回 left（含义：小于 target 的元素有 left 个）

搜索右边界

  • 找到 target 时 left = mid + 1（收缩左边界）
  • 返回 left - 1

二分搜索应用

  1. KoKo 吃香蕉：K 从 1 到 max(piles)，线性遍历 → 二分查找
  2. 货物运输：最低运载能力，范围 [max(weights), sum(weights)]，二分查找

滑动窗口（Sliding Window）

维护一个窗口，不断滑动，更新答案。时间复杂度 O(N)。不停移动右指针扩大窗口，满足收缩条件时移动左指针缩小窗口。

关键问题：left 收缩条件？如何更新答案？结束条件？

经典题目：最小覆盖子串、字符串排列、找所有字母异位词、最长无重复子串。

BFS

问题抽象成图，从起点向四周扩散；使用队列 + visited 集合。每轮取出当前层所有节点，将其未访问的相邻节点入队，step++。与 DFS 相比路径短但空间复杂度大。

  • 二叉树最小高度：root 为 start，叶子为 target
  • 打开密码锁最少次数
  • 双向 BFS 优化：从起点和终点同时开始，交集时停止，只遍历半棵树

nSum 问题

  1. 2Sum：排序 + 双指针；多对不重复时跳过相同元素
  2. 3Sum：穷举第一个数，剩余 2Sum
  3. 4Sum：穷举第一个数，剩余 3Sum
  4. nSum：穷举第一个数，递归 (n-1)Sum

2Sum 核心思想

无序时用哈希表；或在 add 时预计算所有和存入哈希表。

五、设计数据结构

单调栈

栈内元素保持有序。

  1. 下一个更大元素：倒序入栈，栈顶比当前小则弹出，栈顶即为下一个更大元素，最后将自身入栈。
  2. 等几天更暖：存索引，相减得距离
  3. 环形数组：数组长度翻倍 + % 取余

单调队列

队列中元素单调递增或递减。

  • 滑动窗口最大值：双链表实现，入队时删除小于新元素的队尾，头部即最大值。

Union-Find 并查集

解决图的动态连通性问题。连通性：自反性、对称性、传递性。

数组模拟森林，parent[x] 为 x 的父节点，while (parent[x] != x) x = parent[x] 找根。

优化：

  • 按秩合并：小树连大树，额外数组记录节点数
  • 路径压缩：parent[x] = parent[parent[x]] 缩短树高

应用 → 将问题转化为图的动态连通性：

  1. DFS 替代：被 X 围住的 O → 二维转一维 (x,y) → x*n+y，增加虚拟节点 dummy 连接边界 O
    • 方向数组：int[][] d = {{1,0}, {0,1}, {0,-1}, {-1,0}}
  2. 判定合法算式：先遍历 == 建立连通，再遍历 != 检查

LRU（Least Recently Used）

哈希链表。

LFU（Least Frequently Used）

key-freq 表、freq-key 表、key-val 表、变量 minFreq。

Design Twitter Timeline

合并 k 个有序链表（优先级队列）+ 面向对象设计。 API: postTweet, getNewsFeed, follow, unfollow。 用户推文存储在链表（按时间有序），关注 k 个用户 → 合并 k 个有序链表。

六、经典问题

区间问题

排序 + 画图。

  1. 区间覆盖：按起点升序（起点同则降序），三种关系 → 覆盖（计数+1）、相交（合并）、不相交（更新起止）
  2. 区间合并：排序后，相交区间取 min(start) 和 max(end)
  3. 区间交集：交集条件 b2 >= a1 and a2 >= b1，交集 [max(a1,b1), min(a2,b2)]，下标按区间尾大小移动

数组去重

去除重复字母，保持相对顺序，字典序最小：

  • 栈 + 计数器 + 是否在栈中标记
  • 插入时：待插入 < 栈顶 且 栈顶后面还会出现 → 弹出栈顶
