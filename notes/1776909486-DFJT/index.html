算法的真正作用，不是让你在工作中频繁手写高级数据结构，而是训练你写正确代码、看见不变量、识别问题结构，并把系统世界里的抽象连接起来。

在小处持续追求更干净、更短、更对的表达，可能会塑造你在大处做简洁设计的能力（抓住少数关键约束，然后让结构顺下来

  1. 训练写正确小程序：底层逻辑、状态约束、边界条件、复杂度/结构、小正确性、最小失败输入/最小复现路径
  2. 抓性质、不变量：大型成功系统，重要的是10%的架构性质，其他都是胶水和绒毛；不变量 = 在过程推进中始终保持为真的东西；
  3. 识别问题结构：识别 amortized 和 worst-case 的区别；内化核心想法（建立一套结构词汇表）
  4. 知识网（连接所有知识）：数据结构不一定以“原名”出现，但它会作为系统部件反复出现

核心算法清单

  • 查找、排序、存储结构是一条线
    • binary search 到处都是
    • merge sort 和 LSM-tree 连得很近，排序和合并是存储系统的真实基础设施
    • binary heap 到处都是，尤其是 timer、Dijkstra、k-way merge
    • B-tree，对 memory hierarchy 友好，它适配缓存、页、块、磁盘/内存层级
  • 图算法、依赖关系、遍历是一条线
    • DFS / BFS / Topological Sort 非常工程化：DFS：深入探索，BFS：按层扩散，Topological sort：安排依赖执行顺序
    • SCC 是“允许环时”的工具：DAG 没环时，拓扑排序很好办；有环时，就要先理解“环的结构”
    • Floyd-Warshall 是动态规划的“总 Boss”：按阶段维持不变量（逐步放宽可用信息/中间节点集合，让最优子结构不断扩展
  • 字符串、自动机、哈希也是一条线
类别 内容
搜索 linear search, binary search
排序 quadratic sorting, quick sort, merge sort, heap sort
线性结构 growable array (ArrayList/vector), doubly-linked list, skip list
栈/队列 stack, queue, monotonic stack, monotonic queue
树 binary heap, BST, AVL tree, red-black tree, B-tree, splay tree, trie
哈希 hash table (chaining, open addressing)
图遍历 DFS, BFS, topological sort, strongly connected components
图最优 MST (Prim, Kruskal), shortest paths (BFS, Dijkstra, Floyd–Warshall, Bellman–Ford)
字符串 substring search (quadratic, Rabin-Karp, Boyer-Moore, KMP), Aho-Corasick
技巧 two pointers (fast-slow, left-right), sliding window, union-find
范式 divide & conquer, DP (LCS, edit distance), greedy, backtracking, branch & bound
设计 LRU, LFU
