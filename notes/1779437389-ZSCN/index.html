服务间通信的控制平面

服务网格 = 把服务间通信的通用能力，从业务代码/SDK里抽出来，交给基础设施统一处理。

服务发现和负载均衡只是其中两项：解决的是找到谁、连到谁； 服务网格解决的是服务之间如何安全、可靠、可观测、可控制地通信。

istio

通过透明代理把服务间通信收归基础设施管理。

Istio 官方也把架构分成 data plane 和 control plane：

  • 数据平面通常由 Envoy sidecar 组成，负责拦截和控制服务间通信
  • 控制平面 istiod 负责配置这些代理

Istio 通过给每个 Pod 注入 Envoy 代理，劫持进出 Pod 的流量；再由 istiod 统一把路由、安全、负载均衡、熔断、遥测等规则下发给这些 Envoy。 App A -> 本地 Envoy A -> 远端 Envoy B -> App B
