go/types 是 Go 的类型检查器。go/parser 只把源码变成 AST；go/types 负责回答三个语义问题：

  • 这个名字引用哪个声明？
  • 这个表达式是什么类型？
  • 这个常量表达式的值是多少？

三者不能完全分开做：数组长度依赖常量值，unsafe.Sizeof 的值依赖类型，T{k: 0} 里的 k 是字段还是变量又依赖 T 的类型。

基本用法

  • FileSet：把 token.Pos 翻译成文件、行、列。
  • Config：类型检查配置；有 import 时通常要给 Importer。
  • Package：检查后的包，能拿到包名、import path、包级 scope。
  • Info：最重要的结果容器；只有提前初始化的 map 才会被填充。

真实工程更推荐用 golang.org/x/tools/go/packages，它会处理 module、build tags、cgo、依赖和类型检查。

几个核心概念

types.Object 表示“声明出来的东西”：函数、变量、常量、类型名、import 后的包名、内建函数等。同一个声明对应同一个 Object，所以判断“是不是同一个声明”通常可以比较 object 指针。不要用旧的 go/ast.Object，它没有完整类型信息。

types.Info 是 AST 到语义的索引，常用字段：

字段 含义
Defs / Uses identifier 定义/引用了哪个 Object
Types 表达式的类型、常量值、mode
Selections x.f 这种字段/方法选择
Scopes AST block 对应的 Scope

注意：fmt.Println 是 qualified identifier，fmt 和 Println 都看 Uses；只有 x.f 这种字段/方法选择才看 Selections。

Type

types.Type 只有一个核心方法：Underlying() Type。

常见具体类型：Basic、Pointer、Array、Slice、Map、Chan、Struct、Signature、Alias、Named、Interface、TypeParam。

几个经验规则：

  • 不要用 t1 == t2 判断类型相同，用 types.Identical(t1, t2)。
  • 判断赋值、比较、转换、接口实现，用 AssignableTo、Comparable、ConvertibleTo、Implements。
  • 分析底层结构时先用 Underlying()，否则 type MyMap map[string]int 会被当成 Named，而不是 Map：switch typ.Underlying().(type) { ... }。

方法、字段和输出

  • 字段/方法查找不要只用 Name() 做 key；未导出名字还和 package path 有关，用 obj.Id() 或 types.Id(pkg, name)。
  • 获取方法集用 types.NewMethodSet(t)；只查一个字段/方法时用 types.LookupFieldOrMethod 更便宜。
  • Type.String() / Object.String() 常带完整 import path；生成代码或面向用户输出时用 types.TypeString / types.ObjectString 加 qualifier。

使用 checklist

  1. 真实项目优先 go/packages；demo 才手动 parse + check。
  2. 需要什么结果，就初始化 types.Info 对应 map。
  3. ast.Ident 查 Defs / Uses；表达式查 Types；x.f 查 Selections。
  4. 同一声明比较 Object；同一类型用 types.Identical。
