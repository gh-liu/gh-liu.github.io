用二元关系严格描述谁在谁之前

一个序本质上是集合上的二元关系：对元素 a, b，定义 a ⪯ b 表示：a 在某种意义下不晚于 / 不大于 / 先于 b。

我们到底在定义哪种关系？这个关系是否自反、反对称、传递、连接？

偏序全序 Partial / Total Order

偏序：允许两元素「不可比」。只要求有因果/约束的地方有顺序。 全序：任意两元素都能比。任意两个元素都必须排出先后。

≤ 是抽象关系（可以是整除、子集、依赖…，不一定是数值大小）。

** 偏序的三条法则 **

法则 定义 直觉 反例（不满足就不是偏序）
自反 Reflexive a ≤ a 自己跟自己成立 严格小于 <（3<3 不成立）
反对称 Antisymmetric a ≤ b 且 b ≤ a ⟹ a = b 不许「互相超车」=禁止环 「同班同学」（互相同班但不是同一人）
传递 Transitive a ≤ b 且 b ≤ c ⟹ a ≤ c 能一路接力推下去 石头剪刀布的「赢」（赢不传递）

反对称 ≠ 非对称：它允许 a ≤ a，只禁止 a≠b 时的双向箭头。

全序 = 偏序 + 完全性：任意 a,b 必有 a ≤ b 或 b ≤ a。 全序排出来是一条「链」(连接性，所有元素都能排成一条线)，偏序可以分叉。

全序（链）            偏序（可分叉/不可比）
   1                      文件夹
   |                     /      \
   2                  子A        子B   <- A、B 不可比
   |                   |          |
   3                  A1         B1

例子

集合 关系 偏序 全序 说明
整数 ≤ ✅ ✅ 任意两数能比大小
正整数 整除 a∣b ✅ ❌ 2 和 3 互不整除
集合族 子集 ⊆ ✅ ❌ {1} 和 {2} 不可比
字符串 字典序 ✅ ✅ 任意两串都能排序
任务依赖 「先于」 ✅ ❌ 无依赖的任务并行不可比

为什么 CS 里重要

  • 分布式时间：Lamport 时间戳给全序，向量时钟给偏序（能识别并发/不可比事件）；happens-before 本质是偏序。
  • 拓扑排序：把偏序（DAG 依赖）线性化成一个全序。
  • CRDT / 版本向量：用偏序判断冲突（不可比 = 并发修改 = 冲突）。
  • 排序算法：要求全序，否则无法确定唯一排列。
