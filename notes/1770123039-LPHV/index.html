关系型数据库使用SQL作为查询语言，应用程序可以使用相同的SQL查询语句，在不同的数据库中进行查询。

Go语言标准库的database包，提供了一些数据库通用操作的接口。

0.0.1 · 概要

database包有两个子包:

  1. sql: sql包提供对关系型数据库的通用操作API
  2. driver: driver包定义了一系列接口，由特定数据库的驱动实现，然后注册到sql包，再由sql包使用。

原理：抽离数据库驱动，实现driver包中的接口；而用户层代码只使用sql包的类型与方法，sql包调用满足driver包接口的实现即可。

好处：

  1. Go数据类型与DB数据类型转换一致，大部分数据转换在sql包中实现，而不是在特定数据库驱动实现包里。所以数据库驱动只要处理小部分类型集合；
  2. sql包中实现了连接池，是并发安全的，sql.DB可以在多个协程中分享，自动获取底层连接池中的连接，用户代码不需额外的同步原语；
  3. 复杂度放在sql/driver包中，而不暴露给用户

0.0.2 · 简明教程

在Go中通过sql.DB结构体访问数据库，使用sql.DB结构体创建语句和事务，执行查询，并获取结果。

注意：

  1. sql.DB结构体并不是一个数据库连接
  2. 也不是任何特定数据库中__database__或__schema__的概念
  3. 只是数据库操作接口的抽象，其可以是一个本地文件，或者内存数据库，或者是一个网络数据库

0.0.2.1 · 1. 导入一个数据库驱动

使用某个数据库，需要导入特定数据库驱动的实现

import (
	"database/sql"
	_ "github.com/go-sql-driver/mysql"
)

0.0.2.2 · 2. 访问数据库/获取连接

使用sql.open函数创建一个sql.DB值，调用一次即可，后续的操作使用sql.DB完成

func main() {
	db, err := sql.Open("mysql",
		"user:password@tcp(127.0.0.1:3306)/hello")
	if err != nil {
		log.Fatal(err)
	}
	defer db.Close()
}

0.0.2.3 · 3. 查询并获取结果集

func main() {
	var (
		id int
		name string
	)
	rows, err := db.Query("select id, name from users where id = ?", 1)
	if err != nil {
		log.Fatal(err)
	}
	defer rows.Close()
	for rows.Next() {
		err := rows.Scan(&id, &name)
		if err != nil {
			log.Fatal(err)
		}
		log.Println(id, name)
	}
	err = rows.Err()
	if err != nil {
	log.Fatal(err)
	}
}

0.0.2.4 · 4. 修改数据

func main() {
	stmt, err := db.Prepare("INSERT INTO users(name) VALUES(?)")
	if err != nil {
		log.Fatal(err)
	}
	res, err := stmt.Exec("Dolly")
	if err != nil {
		log.Fatal(err)
	}
	lastId, err := res.LastInsertId()
	if err != nil {
		log.Fatal(err)
	}
	rowCnt, err := res.RowsAffected()
	if err != nil {
		log.Fatal(err)
	}
	log.Printf("ID = %d, affected = %d\n", lastId, rowCnt)
}
_, err := db.Exec("DELETE FROM users")

0.0.2.5 · 5. 事务

func main() {
	tx, err := db.BeginTx(ctx, &sql.TxOptions{Isolation: sql.LevelSerializable})
	if err != nil {
		log.Fatal(err)
	}
	id := 37
	_, execErr := tx.Exec(`UPDATE users SET status = ? WHERE id = ?`, "paid", id)
	if execErr != nil {
		_ = tx.Rollback()
		log.Fatal(execErr)
	}
	if err := tx.Commit(); err != nil {
		log.Fatal(err)
	}
}

0.0.3 · driver 包

driver包定义了一系列接口，由特定数据库驱动实现，在使用时先注册到sql包，供sql包使用。

接口定义行为，那么对数据库的操作有哪些行为呢?

  1. 连接数据库
  2. 查询/修改表数据
  3. 事务

0.0.3.1 · 连接数据库

// 数据库驱动需要实现Connector和DriverContext接口
type Driver interface {
	Open(name string) (Conn, error)
}
type DriverContext interface {
	OpenConnector(name string) (Connector, error)
}

// 所有的连接Conn实现，应该实现以下接口，Pinger, SessionResetter, Validator
// 如果数据库支持命名参数或上下文，数据库连接Conn应该实现：ExecerContext, QueryerContext, ConnPrepareContext, ConnBeginTx
type Conn interface {
	Prepare(query string) (Stmt, error)
	Close() error
	Begin() (Tx, error)
}
type Connector interface {
	Connect(context.Context) (Conn, error)
	Driver() Driver
}
// Pinger
// SessionResetter
// Validator
// 一个连接在使用后，在返回连接池之前，如果实现了IsValid ，会被调用。
// 一个连接在其他查询复用前，如果实现了ResetSession，会被调用。
// 一个连接立即复用而不返回连接池时，ResetSession 在复用前调用而IsValid不会被调用。
// ErrBadConn 错误应该只被Validator, SessionResetter接口，或一个查询方法，当连接处于无效的状态。
// ExecerContext, QueryerContext, ConnPrepareContext, ConnBeginTx

0.0.3.2 · 查询/修改表数据

type Stmt interface {
	Close() error
	NumInput() int
	Exec(args []Value) (Result, error)
	Query(args []Value) (Rows, error)
}

type Value any

type Result interface {
	LastInsertId() (int64, error).
	RowsAffected() (int64, error)
}

type Rows interface {
	Columns() []string
	Close() error
	Next(dest []Value) error
}
// 为了支持自定义数据类型，实现NamedValueChecker接口。
// 如果支持多结果集，Rows 应该实现RowsNextResultSet。
// 如果驱动知道如何去描述出现再返回结果中的类型，其应该实现RowsColumnTypeScanType, RowsColumnTypeDatabaseTypeName, RowsColumnTypeLength, RowsColumnTypeNullable, and RowsColumnTypePrecisionScale. 接口。
// 一个给定的value值，同时返回rows类型，代表一个数据库cursor游标值

0.0.3.3 · 事务

type Tx interface {
	Commit() error
	Rollback() error
}

type ConnBeginTx interface {
	BeginTx(ctx context.Context, opts TxOptions) (Tx, error)
}

type TxOptions struct {
	Isolation IsolationLevel
	ReadOnly  bool
}

0.0.4 · sql 包

0.0.4.1 · 注册数据库驱动

func Register(name string, driver driver.Driver)

MySQL 驱动会在go-sql-driver/mysql/mysql.init中调用Register函数注册。

func init() {
	sql.Register("mysql", &MySQLDriver{})
}

0.0.4.2 · 获取 sql.DB，获取数据库连接

// sql包提供的一个db连接池实现，并发安全
// 自动创建/释放db连接
// 维护一个空闲连接池
// DB.Begin调用时候,TX事务绑定一个连接,Commit or Rollback调用时,连接返回空闲池
type DB struct

// 创建DB
// 只验证参数,不创建连接
func OpenDB(c driver.Connector) *DB
// 使用 dsnConnector
// 根据driverName获取驱动
func Open(driverName, dataSourceName string) (*DB, error)

// 在OpenDB中被调用，一个forselect循环，在db.openerCh有值时，调用openNewConnection
func (db *DB) connectionOpener(ctx context.Context)
// 创建一个连接，放入池
func (db *DB) openNewConnection(ctx context.Context)
