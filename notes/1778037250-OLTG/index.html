:h treesitter 总结

2026-05-062026-05-06
CSCLI

Neovim 内置 tree-sitter 增量解析库,用于语法高亮、缩进、折叠、注入语言等。文档主要包括:Parsers / Queries / Highlight / Injections / Defaults / Lua API 几大块。

1. Parsers (解析器) *treesitter-parsers*

  • 解析器是 .so/.wasm 动态库,放在 runtimepath 下的 parser/{lang}.so
  • Nvim 自带:C, Lua, Markdown, Vimscript, Vimdoc, Query
  • 其他语言用 nvim-treesitter 或手动安装
  • 同一语言有多个解析器时,优先级:user config > plugins > bundled
  • 关键 API:
    • vim.treesitter.language.add('python', { path = '...' }) 加载解析器
    • vim.treesitter.language.register('xml', { 'svg', 'xslt' }) 把 filetype 关联到解析器
  • 编译时启用 ENABLE_WASMTIME 后,可加载 .wasm 解析器

2. Queries (查询) *treesitter-query*

  • Lisp 风格的 .scm 文件,放在 queries/{lang}/{purpose}.scm(如 highlights.scm、injections.scm、folds.scm、indents.scm)
  • 通过 pattern + @capture + #predicate / #directive 提取节点信息
  • 默认取 runtimepath 中第一个匹配文件;可用 modeline 控制:
    • ;; inherits: lang1,lang2 —— 继承(包含)其他语言的查询
    • ;; extends —— 与同语言已有 query 合并

常用 Predicates(条件谓词,只查询不改状态)

Predicate 作用
#eq? / #any-eq? 节点文本字符串相等
#match? / #vim-match? 正则匹配(^ $ 锚定节点首尾)
#lua-match? Lua pattern 匹配
#contains? 包含子串
#any-of? 多选一(关键字推荐用,做了优化)
#has-ancestor? 祖先节点类型匹配
#has-parent? 直接父节点类型匹配
#not-xxx? 任何谓词都可加 not- 前缀取反
#any-xxx? 量化捕获时,任一节点满足即可(否则全部)

可通过 vim.treesitter.query.add_predicate() 扩展, vim.treesitter.query.list_predicates() 列出全部。

常用 Directives(指令,带副作用,设置 metadata)

Directive 作用
#set! 设置 metadata(如 priority、conceal、bo.commentstring)
#offset! 偏移节点范围 {srow, scol, erow, ecol}(常用于 injection)
#gsub! 用 Lua pattern 改写节点文本(metadata.text)
#trim! 去掉首尾空白/空行(常用于 fold、injection 边界)

可通过 vim.treesitter.query.add_directive() 扩展。

3. 语法高亮 *treesitter-highlight*

  • 通过 highlights.scm 把节点映射到 @capture,@capture 直接作为 highlight group
  • 启用方式:vim.treesitter.start()(假设有解析器和 query)
  • 回退机制:@comment.documentation → @comment → Comment
  • 按语言细分:hi @comment.lua guifg=DarkBlue、hi link @comment.documentation.java String

标准 Captures(节选)

变量/常量:  @variable[.builtin|.parameter|.member]  @constant[.builtin|.macro]
模块/标签:  @module[.builtin]  @label
字符串:    @string[.documentation|.regexp|.escape|.special[.symbol|.path|.url]]
字符/数字:  @character[.special]  @boolean  @number[.float]
类型:      @type[.builtin|.definition]  @attribute[.builtin]  @property
函数:      @function[.builtin|.call|.macro|.method[.call]]  @constructor  @operator
关键字:    @keyword[.coroutine|.function|.operator|.import|.type|.modifier
                  |.repeat|.return|.debug|.exception|.conditional[.ternary]
                  |.directive[.define]]
标点:      @punctuation.{delimiter,bracket,special}
注释:      @comment[.documentation|.error|.warning|.todo|.note]
Markup:    @markup.{strong,italic,strikethrough,underline,heading[.1-6],
                    quote,math,link[.label|.url],raw[.block],list[.checked|.unchecked]}
Diff/标签:  @diff.{plus,minus,delta}   @tag[.builtin|.attribute|.delimiter]

特殊 Captures / Metadata

  • @spell / @nospell —— 配合内置 spell 拼写检查
  • @conceal / @noconceal —— 隐藏文本(#set! conceal "x" 单字符替换)
  • #set! conceal_lines "" —— 整行隐藏(尊重 conceallevel / concealcursor)
  • #set! priority 105 —— 调整高亮优先级(默认 100,基于 nvim_buf_set_extmark)
  • #set! @_tag bo.commentstring "{/* %s */}" —— 节点级 commentstring(JSX 等场景),内层 wins

4. 语言注入 *treesitter-language-injections*

通过 injections.scm 让一个文件混合多种语言(HTML 中的 JS/CSS、Lua 中的 vim.cmd、Vimscript 中的 lua-heredoc 等)。

Capture / 属性 作用
@injection.content 用其它语言重新解析该节点的内容(多个会合并为一棵树)
@injection.language 该节点文本是语言名
@injection.filename 该节点文本是文件名,通过 vim.filetype.match() 推断语言
injection.language 属性 硬编码语言名
injection.combined 所有匹配节点合并解析为一个文档
injection.include-children 重新解析时包含子节点文本(默认排除)
injection.self 与节点所在 LanguageTree 同语言
injection.parent 与父 LanguageTree 同语言

注入查询会跑全 buffer,大文件可能慢。要禁某语言注入,放空文件 queries/{lang}/injections.scm。

5. 默认键位 *treesitter-defaults*

可视模式下的增量选区(无 TS 解析器时退化为 vim.lsp.buf.selection_range()):

键 作用
an 选第 [count] 个父节点
in 选第 [count] 个前/首个子节点
]n / [n 跳到下/上一个节点
]N / [N 扩展选区到下/上一个节点

6. Lua API *lua-treesitter*

后半部分(~1200 行)是自动生成的 API 文档,核心模块:

  • vim.treesitter
    • start(bufnr, lang) / stop(bufnr) —— 开关高亮
    • get_parser() / get_node() / get_range() / get_captures_at_cursor()
    • inspect_tree()(:InspectTree)、:EditQuery、:Inspect
  • vim.treesitter.language:add(), register(), inspect()
  • vim.treesitter.query:parse(), get(), add_predicate(), add_directive(), list_predicates(), list_directives()
  • 对象类型:
    • TSTree —— 解析后的语法树
    • TSNode —— 节点遍历:parent / child / next_sibling / prev_sibling / ...、 range()、type()、field()、named_child()、iter_children()
    • LanguageTree —— 由注入构成的解析器树(根 + 子语言)
    • Query —— 编译后的查询对象,可 iter_matches() / iter_captures()

调试小抄

命令 作用
:InspectTree 打开 AST 可视化窗口
:Inspect 看光标处的高亮组及来源(TS/syntax)
:EditQuery 交互式编辑/调试 query
:checkhealth treesitter 检查解析器/版本兼容性
