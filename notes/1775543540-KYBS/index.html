排障

https://darryl-ruggles.cloud/kubectl-commands-i-actually-use-every-day-a-practical-guide-to-kubernetes-debugging-and-operations

kubectl 本质上只是读取 ~/.kube/config，然后向 API Server 发 HTTP 请求

如果kubectl慢、超时、失败时，很多时候该怀疑的是到 API Server 的网络链路或权限。

思路：看状态 → 看事件 → 看日志 → 进容器 → 修复

  1. Assess：先看当前状态
  2. Investigate：再查原因
  3. Interact：必要时进容器里看
  4. Resolve：最后处理问题
  • Pod 不工作？状态异常？k get pods
  • 刚刚发生了啥？k get events --sort-by='.lastTimestamp'
  • 资源问题？
  • 深入Pod细节 k describe pod
  • 看日志 stern or k logs (--previous?)
  • 确认内部 k exec
