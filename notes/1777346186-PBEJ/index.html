缓存

S 什么场景 C 什么冲突 Q 什么问题 A 回答问题

把常用数据放到更快、更近的地方，减少重复计算和数据库访问。

  1. 降低数据库读压力
  2. 降低延迟
  3. 抗热点流量
  4. 保存临时状态
  5. 保存预计算结果

数据有两份之后，如何保证一致性？

  1. Cache Aside 旁路缓存：读请求先查缓存，未命中再查数据库并回填；写请求先更新数据库，再删除缓存，让后续读请求重新加载。

缓存一致性通常不是绝对强一致，而是：尽量缩短不一致窗口。 缓存一般用于可容忍短暂不一致的读场景，通过删除缓存、TTL、重试和异步补偿来控制不一致窗口。

缓存穿透、击穿、雪崩

  • 缓存穿透 Cache Penetration：查询一个缓存和数据库都不存在的数据，导致每次都打到数据库
  • 缓存击穿 Cache Breakdown：一个热点 key 过期瞬间，大量请求同时打到数据库
  • 缓存雪崩 Cache Avalanche：大量 key 同时过期，或者 Redis 故障，导致流量全部打到数据库
