EXPLAIN

字段 看什么
type 访问方式，是否退化成 ALL
key 实际用了哪个索引
rows 估计扫描多少行
Extra Using index（覆盖索引）、Using filesort、Using temporary

慢查询排查

开启 slow_query_log → mysqldumpslow 聚合 → EXPLAIN 看执行计划 → 优化索引/SQL/表结构

performance_schema

MySQL 内置的运行时诊断工具，可查：等待事件（锁等待、IO 等待）、SQL 统计（执行次数、耗时分布）、内存使用。

备份与恢复

工具 类型 特点
mysqldump 逻辑备份 简单，但大库慢
xtrabackup 物理备份 快，支持增量备份，不锁表（InnoDB）

恢复流程：全量备份还原 → 追加 binlog 到目标时间点（PITR）。
