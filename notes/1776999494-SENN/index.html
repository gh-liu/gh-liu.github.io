ip 是 Linux 中查看和管理网络配置的核心命令，常用来统一替代旧的 ifconfig、route、arp。

ip 子命令和网络分层

ip 的子命令不是严格按 OSI 分层设计的，但可以这样理解：

子命令 对应概念 更接近哪层 作用
ip link 网络接口、MAC、MTU、接口状态 二层（兼带一点一层状态） 查看/操作网卡和链路状态
ip neigh 邻居表、ARP/NDP 二层和三层之间 查看 IP 到 MAC 的映射
ip addr 本机 IP 地址、子网前缀 三层 查看/配置网卡上的 IP 地址
ip route 路由表、下一跳、默认网关 三层 查看/配置数据包该往哪走

可以记成一句话：

  • link：我有哪些口
  • addr：我是谁
  • route：我要往哪走
  • neigh：下一跳是谁的 MAC

ip link

用于查看和操作网络接口。

它回答的问题是：

  • 机器上有哪些网卡？
  • 网卡是否启用？
  • MAC 地址是什么？
  • 链路是否正常？

ip neigh

用于查看邻居表，也就是 IP 到 MAC 的解析结果。

它回答的问题是：

  • 某个 IP 对应的 MAC 地址是什么？
  • ARP/NDP 解析是否正常？

ip addr

用于查看和配置接口上的 IP 地址。

它回答的问题是：

  • 这块网卡的 IP 是什么？
  • 它属于哪个子网？

ip route

用于查看和配置路由表。

它回答的问题是：

  • 发往某个目标 IP 的包会从哪个接口出去？
  • 下一跳是谁？
  • 默认网关是谁？

ip rule 和 ip route 的关系

ip rule 也和路由有关，但它管理的不是某张路由表里的具体路由，而是“什么包去查哪张路由表”。

  • ip rule：选表
  • ip route：填表
