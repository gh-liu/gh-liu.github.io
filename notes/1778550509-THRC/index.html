工程方向分为：计算、存储、网络、操作系统（系统综合）；

对应四个底层能力：

方向 关注什么 对后端/系统设计的意义
计算 Compute 并行计算、高性能计算、调度、资源利用率 理解批处理、流处理、调度器、云计算资源管理
存储 Storage 文件系统、KV、日志、LSM、复制、持久化 理解 MySQL、Redis、Kafka、对象存储、日志系统
网络 Network 协议、拥塞控制、数据中心网络、RPC、负载均衡 理解微服务、Service Mesh、CDN、分布式通信
操作系统 / 系统综合 OS 进程、调度、隔离、容错、分布式系统、资源管理 理解 Kubernetes、数据库、分布式系统的底层机制
  • OS: OSDI / SOSP
    • 理解系统为什么这样设计
    • 知道系统未来会长什么样
  • Storage: FAST
  • Network: SIGCOMM / NSDI
  • 工业界：公司Google,Microsoft等；生态CloudNative,Infra等
