Go: 类型与值 types&values

2026-07-092026-07-091 出链1 引用
CSGO

Go 的类型规则围绕三层关系展开：

  1. 表示方式决定赋值是否共享底层数据
  2. 底层类型决定哪些类型在结构上相容
  3. 类型同一性/可赋值性/方法集决定编译器允许哪些赋值、转换、接口实现和方法调用

值的表示

  • 自包含值：预声明类型、数组、结构体的值包含自身全部数据；变量直接存储整个值。它们的零值由类型决定，永远不是 nil。
  • 引用型值：非 nil 的 pointer、function、slice、map、channel 值包含对底层数据的引用，多个值可能共享同一份底层数据。
    • pointer：引用某个变量。
    • function：引用函数体以及闭包捕获变量。
    • slice：包含长度、容量、底层数组引用。
    • map/channel：引用运行时实现的数据结构。
  • interface 值：存储 dynamic type 和 dynamic value；dynamic value 的表示方式取决于 dynamic type。若 dynamic type 是 int、array、struct 等自包含类型，则值自包含；若是 slice、map、pointer、function、channel 等引用型类型，则包含对底层数据的引用。

nil 是可能包含引用的类型的零值：pointer、function、slice、map、channel、interface。

赋值只替换变量中存储的数据；如果存储的是引用，则复制引用而不是复制底层数据。因此修改共享底层数据会影响其他值，例如多个 slice 共享同一底层数组。

底层类型 underlying type

每个类型 T 都有一个底层类型：

  • 预声明的 boolean、numeric、string 类型，以及类型字面量，其底层类型就是自身。
  • 其他类型的底层类型，是其声明中所引用类型的底层类型。
  • 类型参数的底层类型是它的类型约束；约束总是 interface。
  • A1、B1、B2 的底层类型都是 string。
  • B3、B4 的底层类型都是 []B1，不是 []string；因为 slice 字面量的元素类型是 B1。

类型同一性 type identity

两个类型要么 identical，要么 different。

  • 命名类型 named type 总是不同于其他任何类型；两个不同的 type definition 会创建不同的命名类型。
  • 非命名类型如果底层类型字面量结构等价，则 identical：
    • array：元素类型 identical，长度相同。
    • slice：元素类型 identical。
    • struct：字段顺序、字段名、字段类型、tag、是否 embedded 都相同；不同包的未导出字段名总是不同。
    • pointer：基类型 identical。
    • function：参数和结果数量、对应类型、是否 variadic 相同；参数名和结果名不要求相同。
    • interface：定义相同 type set。
    • map：key 和 element 类型 identical。
    • channel：element 类型和方向相同。
    • instantiated type：defined type 和所有 type argument identical。

例子：

  • A0 与 []string identical，因为 A0 是 alias。
  • B0 与 B1 different，因为它们来自不同的 type definition。
  • B0 与 []string different，因为命名类型不同于类型字面量；但在满足规则时可能 assignable。

可赋值 assignability

值 x 的类型为 V，变量类型为 T。满足任一条件时，x 可赋值给 T：

  • V 和 T identical。
  • V 和 T 底层类型 identical，二者都不是类型参数，且至少一个不是 named type。
  • 二者是 channel，元素类型 identical，V 是双向 channel，且至少一个不是 named type。
  • T 是非类型参数的 interface，且 x implements T。
  • x 是 nil，且 T 是 pointer、function、slice、map、channel、interface，但不是类型参数。
  • x 是 untyped constant，且可由 T 表示。

涉及类型参数时，还要求对 type set 中的每个类型都成立。直觉上：泛型赋值必须对约束允许的所有具体类型都安全。

常量可表示性 representability

常量 x 可由类型 T 的值表示，意思是编译期常量能安全落入 T 的值集合：

  • x 属于 T 的值集合，例如 1024 可由 int16 表示。
  • T 是浮点类型，x 可按 T 精度舍入且不溢出；舍入遵循 IEEE 754 round-to-even，负零会进一步简化为无符号零。
  • T 是复数类型，real(x) 和 imag(x) 都可由其分量类型表示。
  • 如果 T 是类型参数，x 必须可由 T 的 type set 中每个类型表示。

典型反例：1024 不能由 byte 表示，-1 不能由 uint16 表示，1.1 不能由 int 表示，1e1000 不能由 float64 表示。

方法集 method set

Go: 方法集 method sets：方法集决定某个类型的操作数能调用哪些方法。
