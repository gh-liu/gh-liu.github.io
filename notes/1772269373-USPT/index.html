go 命令可以使用内置工具链，也可以使用PATH中的其他版本（按需下载）

通过如下控制：

  1. go.mod或go.work中的go或toolchain指令
  2. GOTOOLCHAIN 环境变量：默认auto，按go/toolchain选择；local始终内置；go1.21.0固定某个版本；go1.21.3+auto默认但允许按模块要求升级
配置项 作用
go 模块/工作区最低版本
toolchain 建议使用的工具链版本
GOTOOLCHAIN 强制指定使用的工具链版本

go version

作用：查看 Go 二进制文件的构建信息

  1. 查看当前Go版本: go version
  2. 指定二进制，查看构建信息: go version -m <file>

参数：

  1. -m: 模块/依赖/构建信息
  2. -json: 配合-m使用，输出json

go get

go get 用于管理模块依赖

go list

列出指定包或模块的信息

go build

编译指定包及其依赖

go doc

查看 Go 包、类型、函数、方法等的文档注释

go install

编译并安装指定包

  • 带版本：go install pkg@version 按指定版本独立安装 main 包(目标go.mod不能有replace/exclude歧义)，忽略当前项目 go.mod，适合安装工具。
  • 不带版本：go install pkg / go install ./cmd/x 在当前 module/GOPATH 上下文里解析并安装，能否成功取决于当前上下文是否能解析该包。

编译工具与使用示例

工具链概览

工具 功能 阶段
go tool compile 编译源文件为目标文件 (.o/.a) 前端+中端
go tool asm 汇编器，编译汇编代码 后端
go tool link 链接器，链接目标文件为可执行文件 后端
go tool objdump 反汇编工具，查看二进制内容 分析
go tool nm 符号列表工具 分析
go tool cgo C 语言互操作 特殊
go build 高级构建工具（自动调度） 完整流程
