1 · interface 接口#

https://go.dev/ref/spec#Interface_types

https://research.swtch.com/interfaces

An interface type defines a type set.

接口类型定义了一个类型集。 接口类型值，可以存储类型集中任一类型的值，是一种和类型。

未初始化过的接口类型的值是nil

在没有泛型时，这个类型集，可以看作一组方法的签名，所有满足这些方法的类型，都属于这个类型集。 这是Go接口的优点之一：隐式接口，实现了接口的所有方法的类型就隐式地实现了接口。

有泛型后，接口承担起对类型的约束，这个类型集(多个类型的并集)，就是约束了类型的范围。 类型集中可以是具体类型TypeTerm，也可以是底层类型~Type。详细见泛型。

接口与具体实现分离, 依赖一个约定好的接口

传递参数、返回参数以及变量赋值时才会对某个类型是否实现接口进行检查

类型: 出现在变量的定义、函数的入参和返回值中并对它们进行约束； 一种是带有一组方法的接口，另一种是不带任何方法的 interface{}

1.1 · 静态、动态

Go 的接口是静态的（编译时检查）和动态的（运行时查询）的结合体。

静态：赋值、泛型约束 动态：类型断言

1.2 · 接口类型值在运行时的底层表示

接口值由两个字组成：

┌─────────────────┬─────────────────┐
│   itable 指针   │    数据指针     │
└─────────────────┴─────────────────┘

itable 包含：

  1. 类型元数据
  2. 函数指针列表

重要特性: itable 对应的是接口类型，而非动态具体类型。

// runtime.iface
type iface struct {
	tab  *itab // 重要
	data unsafe.Pointer // 原始数据
}

// 插一句 statis type/ concreate type/ interface type
type itab struct {  // 核心， 接口类型与具体类型的组合，分别inter与_type， 哈希快速判断与_type是否一直，fun是动态大小数组，虽固定大小，使用原始指针获取获数据
	inter *interfacetype
	_type *_type
	hash  uint32 // copy of _type.hash. Used for type switches.
	_     [4]byte
	fun   [1]uintptr // variable sized. fun[0]==0 means _type does not implement inter.
}

1.2.1 · 特殊 interface#

对于 interface{}（无方法），itable 只用于指向类型，可以省略：

┌──────────────┬──────────────┐
│  类型指针    │  数据指针    │
└──────────────┴──────────────┘
// runtime.eface 在 Go 语言中很常见，实现时使用了特殊的类型
type eface struct {
	_type *_type // 底层数据类型
	data  unsafe.Pointer // 底层数据
}

1.3 · 方法动态派发

调用接口的方法，在编译期间不能确定接口的类型，那么在运行期间决定调用该方法的哪个实现；

接口类型调用，运行时动态派发； 具体类型调用，编译期确定；使用 -N 关闭编译器优化

获取 itab->fun[0] 到寄存器，iface 的 data 作为接收者参数放到寄存器，通过 CALL 间接调用

1.3.1 · 例子

// dispatch.go
package main

type Stringer interface {
    String() string
}

type Binary uint64

func (b Binary) String() string {
    return "binary"
}

func main() {
    var s Stringer = Binary(42)
    s.String() // 动态派发
}
go tool compile -S -N dispatch.go
; 核心部分：动态派发实现
main.Stringer.String STEXT dupok size=130 args=0x10 locals=0x30
    ...
    0x0021  MOVQ    main.~p0+56(SP), CX       # CX = tab (itable 指针)
    0x002c  TESTB   AL, (CX)                  # nil 检查
    0x002e  MOVQ    main.~p0+64(SP), AX       # AX = data            (设置接收者参数)
    0x0033  MOVQ    24(CX), CX                # CX = itable->fun[0]  (获取函数指针)
    0x0037  CALL    CX                        # 调用 (*Binary).String(AX)
    ...
寄存器内容
AXitable 指针
BXdata 指针
  1. 获取 tab.func[0]: MOVQ 24(AX), CX - 从 itable 偏移 24 字节处读取函数指针
  2. 数据拷贝: MOVQ BX, AX - 将接口的 data 字设置为接收者参数
  3. 间接调用: CALL CX - 通过函数指针动态调用实际方法

Go 将方法查找从每次调用移到了类型转换时，itable 缓存后，后续调用只需间接跳转。

调用链：

main.main
    │
    ├─ var s Stringer = Binary(42)  // 构造接口值
    │
    └─ s.String()  // 动态派发
           │
           ↓
    main.Stringer.String  (编译器自动生成的包装器)
           │
           ├─ MOVQ 24(AX), CX  # 获取 itable->fun[0]
           ├─ MOVQ BX, AX      # 设置接收者
           └─ CALL CX          # 间接调用
                  │
                  ↓
    main.(*Binary).String  (实际实现)
           │
           ├─ LEAQ go:string."binary"(SB), AX
           ├─ MOVL $6, BX
           └─ RET
