信息论

2026-05-132026-05-13 16:36:05
CS

研究"信息"能不能被测量、能压到多小、能传多远而不出错

香农 1948 年问了三个朴素问题:

  1. 一条消息到底"含多少信息"? → 能不能测量，用数字表示?
  2. 文件能压到多小? → 有没有极限?
  3. 网络有噪声,还能传对吗? → 有多大带宽?

他给出的答案,就是今天的信息论。信息论 = 把"惊讶"变成数字,再用这把尺子去衡量压缩、学习、传输、沟通中的一切权衡。

记住"信息 = 消除的不确定性"这一句,剩下的全是它的推论。信息不是内容本身，而是内容带来的不确定性下降。

最关键的一个直觉：信息 = 意外

越"出乎意料"的事,信息量越大；概率越小 → 信息量越大。

举例:

  • "明天太阳从东边升起" → 你早知道,信息量≈0
  • "明天公司放假一周" → 有点意外,信息量大
  • "你中了五百万" → 极度意外,信息量爆炸

所以信息量不是看"字数多少",而是看"概率多小"。

公式：

某事件的信息量 = log2(1 / 概率)   单位: bit

五个最常用的"尺子"(工具)

统一故事:你和朋友玩"二十问"——他心里想一个东西,你只能问"是不是 XX?"这种是非题,目标是用最少的问题猜出来。 五个工具,就是五种不同情境下"该问几次"的衡量。

先看一张总览,再逐个解释:

尺子 一句话 类比"二十问" 工程里就是…
熵 最佳策略下平均要问几次 一个最聪明的人最少问几次能猜到 压缩极限/不确定性
交叉熵 你用了错策略,平均要问几次 你猜错了重点,问了一堆冤枉问题 模型 loss
KL 散度 你"多花"了几次 比最聪明的人多问的次数 两个分布的差距
互信息 知道一条线索后能少问几次 朋友偷偷告诉你"是吃的" 特征有没有用
信道容量 嘈杂房间里每秒最多能传几个答案 朋友说话有杂音,一秒只能听清几个字 带宽上限

1. 熵 H —— 「最少要问几次」

场景:朋友从 4 种水果(苹果/香蕉/橘子/葡萄)中等概率选一个。 最佳策略:先问"是苹果或香蕉吗?",再问一次细分 → 2 次搞定。 → 熵 = log2(4) = 2 bit。

关键直觉:

  • 选项越多 / 越均匀 → 熵越大 → 越难猜。
  • 如果朋友 99% 选苹果 → 你直接猜苹果几乎都对 → 熵接近 0。

工程含义:

  • 一个文件的熵 = 它最多能压到多小(ZIP 极限)。
  • 一个密码的熵 = 攻击者平均要试多少次。

2. 交叉熵 —— 「你用错策略,要问几次」

接上例:朋友其实 99% 时间选苹果,但你不知道,以为四种水果均等。 你的策略:每次都先排除一半 → 平均还是问 2 次。 最佳策略:第一句就猜"是苹果吗?" → 平均不到 1 次。

你"用错了世界观",所以多问了 → 这个实际平均次数就是交叉熵。

一句话:用 q 的策略去猜 p 的世界,平均代价。

工程含义:

  • 训练一个分类器 = 让模型的"猜测分布 q"去逼近"真实分布 p"。
  • loss = 交叉熵 = 模型猜错的代价。loss 下降 = 模型越来越懂世界。

3. KL 散度 —— 「比最聪明的人多问几次」

接上例:

  • 最佳策略平均 0.1 次(熵)
  • 你的策略平均 2 次(交叉熵)
  • 多花的 1.9 次,就是 KL 散度。

公式上就是:KL = 交叉熵 − 熵。

一句话:两个分布差得有多离谱。

工程含义:

  • 模型训练目标 = 把 KL 压到 0(贴近真实)。
  • 线上监控:今天的数据分布 vs 上线时的分布,KL 突然变大 → 数据漂移告警。

4. 互信息 —— 「提示能让你少问几次」

场景:还是猜水果,但朋友先告诉你"今天是黄色的"。

  • 没提示前:要问 2 次。
  • 知道颜色后:只剩香蕉 → 0 次。
  • 节省的 2 次 = "颜色"和"水果"之间的互信息。

一句话:知道 A 之后,对 B 的不确定性减少了多少。

关键直觉:

  • 提示完全无关(比如"火星温度") → 一次都省不了 → 互信息 = 0。
  • 提示就是答案本身 → 全省了 → 互信息 = 熵。

工程含义:

  • 加一个新特征值不值? → 看它和标签的互信息有没有 > 0。
  • 推荐系统:你点过的商品和待推商品的互信息越高,越可能感兴趣。

5. 信道容量 —— 「嘈杂下每秒最多传几次答案」

场景:朋友远远地喊答案给你听,但中间有噪音,偶尔听错。

  • 安静房间:他每秒说 5 个字,你全听清 → 容量 5 字/秒。
  • 嘈杂房间:他还得重复、用拼音、用编码 → 真正传到你耳朵里的有效信息变少。
  • 不管他怎么努力,有个上限——这就是信道容量。

一句话:带噪声的通道,每单位时间最多能可靠传多少 bit。

工程含义:

  • WiFi 信号差时网速变慢:不是机器没努力,是触到了香农极限。
  • 二维码弄脏一块还能扫:故意加冗余去抗噪,代价就是减少容量。
  • 硬盘 ECC 纠错码同理。

思维迁移

学完信息论,你看世界会多三副眼镜:

  • 眼镜 1:「信息量 = 意外」—— 表达与沟通：只说对方没料到的部分,已知的就是噪声。
场景 信息论视角
写代码注释 写"读者意外的部分"才有信息量,废话 = 0 bit
做 PPT 把听众已知的删掉,留高信息量内容
沟通 对方的先验越接近你,需要说的话越少
  • 眼镜 2:「熵 = 不确定性」—— 学习与决策：主动去啃高熵的部分,信息增益最大。
场景 信息论视角
学习 做不会做的题(高熵)才涨知识,刷会的题信息量低
调 bug 二分法 = 每次问一个把可能性砍一半的问题,正是熵最优策略
做选择 优先做能消除最多不确定性的那一步
  • 眼镜 3:「信道 = 噪声 + 带宽」—— 传输与协作：任何沟通都是带噪信道,要么压缩、要么纠错、要么扩容。
场景 信息论视角
跨团队协作 加术语表/规范 = 给信道编码,降低误解率
文档/接口设计 冗余适度,既能纠错又不超过带宽
监控告警 信噪比太低的告警等于没有
