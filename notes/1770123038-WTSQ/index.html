实现原理

Go 泛型提供的是编译期类型参数化，不保证总是 Rust/C++ 那种完全静态分发；

Go 泛型实现采用 dictionary + GC shape stenciling，是性能、编译速度、二进制大小之间的折中。

Go 泛型实现 = GC Shape Stenciling + Dictionaries，介于「完全单态化」与「完全装箱」之间。

1. GC Shape Stenciling（按 GC 形状模板化）

编译器不为每个具体类型都生成一份代码（C++/Rust 那样会让二进制爆炸），而是按 GCShape 分组生成：

  • 同一 GCShape <-> 在 GC 看来内存布局相同（大小、指针/非指针位图一致）
  • 所有指针类型共享一份代码（*int、*string、*T 都是一个 word + 是指针）
  • 值类型按布局分组：int32 与 float32 同形状、int64 与 float64 同形状
  • 结果：N 个实例化类型 → 只生成「去重后 GCShape 数量」份机器码（≤ N，常常远小于 N）

举例：func F[T any](x T) 被以下类型实例化：

F[*User]    ┐
F[*Order]   ├─→ 都是 ptrshape，共享 1 份代码
F[*int]     ┘
F[int64]    ┐
F[float64]  ┴─→ 都是 int64shape，共享 1 份代码
F[string]   ──→ stringshape，1 份代码

N=6 个类型 → 只生成 3 份机器码（3 个不同的 GCShape）。

2. Dictionary（字典传递）

同形状的不同类型语义不同（方法调用、new(T)、==、reflect 需要具体类型信息），编译器在每个泛型函数调用处隐式传入一个 dict 参数：

心智模型：同一份函数代码 + 调用者按调用点传不同的"类型说明书 (dict)" + 函数内部对类型敏感的操作都改写成"从 dict 里取指针/调函数"。

编译期:                            运行期:
F[*User](u)  ─生成─▶ dict_FU       F_ptrshape(dict, x){
F[*Order](o) ─生成─▶ dict_FO         x.Less(y)
                                      ↓ 编译为
同一份 F_ptrshape 代码                dict.methods[0](dict, x, y)
不同调用点选不同 dict               }

不是类型断言，是 dict 传参

容易误解为"函数内部 switch x.(type)"，实际不是：

维度 类型断言（错误模型） Dict 传参（实际做法）
类型信息从哪来 值里带（如 interface{} 的 itab） 编译期在调用处生成的字典
值的表示 装箱（带 type header） 保持原样（裸 *User、裸 int64）
分派开销 运行时 switch / itab 查表 直接从 dict 偏移取指针，一次间接调用
谁选类型 函数运行时自己判断 调用者编译期就把类型"打包"进 dict

dict 用于"类型层面的元信息和方法"，不用于字段访问

Go 泛型没有"结构化类型"约束（不像 TypeScript），所以根本没有"通过 dict 访问字段"这回事——dict 只服务于类型元信息和方法分派。

3. 三种实现策略对比

策略 性能 编译速度 二进制大小
完全单态化 (C++/Rust) 最快 慢 大
完全装箱 (Java 早期) 慢（装箱） 快 小
Go: shape + dict 中等 中等 中等

4. 性能含义

  • 接口约束的方法调用走 dict → 间接调用，无法内联，比手写代码慢
  • 同形状但不同类型 → 共享代码，但每次调用要查 dict
  • 指针类型最受益（共享多）；值类型仍有近似 boxing 的开销

5. 简化示意

源码:                       编译后:
func F[T any](x T) {...}    F_ptrshape(dict, x *any)    ← 所有指针 T 共享
                            F_int64shape(dict, x i64)   ← int64/float64 共享

调用 F[*User](u):
  dict_F_User = { typ:*User, ... }
  call F_ptrshape(&dict_F_User, u)

官方设计文档：https://github.com/golang/proposal/blob/master/design/generics-implementation-gcshape.md
