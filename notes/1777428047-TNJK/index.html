GPU 不是更快的 CPU，而是为高并行吞吐设计的另一种机器。

维度 CPU GPU
优化目标 低延迟 高吞吐
擅长 分支复杂、顺序逻辑、通用任务 大规模并行、矩阵/向量计算
硬件重点 cache、control、branch prediction 大量 ALU / cores
类比 一个强专家 一大群工人

为什么 GPU 会快

前提不是单个核心更强，而是同一时刻能让更多简单计算一起做。

  1. CPU 把很多晶体管花在控制逻辑上，比如乱序执行、分支预测、超大缓存。
  2. GPU 把更多面积花在算术单元上，减少为单线程服务的豪华配置。
  3. 如果一个任务能拆成海量、相似、彼此独立的小计算，GPU 就能把这些计算铺开并行执行。

所以 GPU 的优势本质上是：

牺牲单任务灵活性，换取海量任务总吞吐。

GPU 的代价

GPU 并不擅长所有问题。它通常会在下面几种情况下吃亏：

  1. 分支很多。 同一批线程如果走不同路径，会互相等待，降低并行效率。
  2. 数据依赖强。 上一步没算完，下一步就不能开始，这类串行链条很难喂饱 GPU。
  3. 数据量太小。 把数据搬到 GPU、启动 kernel 都有成本，小任务常常不划算。
  4. 内存访问不规律。 线程读写地址太散，会让带宽利用率变差。

一个更准确的心智模型

  • CPU: 追求把一个任务尽快做完。
  • GPU: 追求单位时间内把尽可能多的任务做完。

所以判断一个任务适不适合 GPU，不要先问它算得多不多，而要先问：

它能不能被拆成很多份、并且每一份都差不多、能同时做？

典型适用场景

  • 矩阵乘法
  • 向量运算
  • 图像处理
  • 深度学习训练与推理
  • 大规模数值模拟

这些任务的共同点不是高级，而是重复、规则、可并行。
