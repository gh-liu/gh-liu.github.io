2. ж ёеҝғеҜ№иұЎ

Provider вҖ” иҝһжҺҘе…·дҪ“е№іеҸ°

Terraform жң¬иә«дёҚжҮӮйҳҝйҮҢдә‘гҖҒAWSгҖҒGCPгҖҒKubernetesпјҢе®ғйҖҡиҝҮ Provider жҸ’д»¶ е’ҢеӨ–йғЁзі»з»ҹдәӨдә’гҖӮ

  • hashicorp/awsпјҡз®ЎзҗҶ AWS иө„жәҗ
  • aliyun/alicloudпјҡз®ЎзҗҶйҳҝйҮҢдә‘иө„жәҗ
  • hashicorp/kubernetesпјҡз®ЎзҗҶ Kubernetes иө„жәҗ
  • hashicorp/helmпјҡз®ЎзҗҶ Helm Release

дҪ еҸҜд»ҘжҠҠ Provider зҗҶи§ЈжҲҗвҖңжҹҗдёӘе№іеҸ°зҡ„ API йҖӮй…ҚеҷЁвҖқгҖӮ

Resource вҖ” Terraform з®ЎзҗҶзҡ„иө„жәҗ

Terraform жңҖж ёеҝғзҡ„еҜ№иұЎе°ұжҳҜ resourceгҖӮ

еҗ«д№үпјҡ

  • aws_instanceпјҡиө„жәҗзұ»еһӢ
  • webпјҡжң¬ең°еҗҚеӯ—
  • еқ—йҮҢзҡ„еӯ—ж®өпјҡиҝҷдёӘиө„жәҗзҡ„жңҹжңӣеұһжҖ§

Terraform дјҡжҠҠжҜҸдёӘ resource зңӢжҲҗзҠ¶жҖҒеӣҫдёӯзҡ„дёҖдёӘиҠӮзӮ№пјҢе№¶ж №жҚ®еј•з”Ёе…ізі»жҺЁеҜјдҫқиө–йЎәеәҸгҖӮ

Data Source вҖ” иҜ»пјҢдёҚеҲӣе»ә

жңүдәӣдҝЎжҒҜдёҚжҳҜдҪ еҲӣе»әзҡ„пјҢиҖҢжҳҜвҖңжҹҘеҮәжқҘеҶҚеӨҚз”ЁвҖқпјҢиҝҷж—¶з”Ё dataпјҡ

еҢәеҲ«пјҡ

  • resourceпјҡеҲӣе»әжҲ–з®ЎзҗҶеҜ№иұЎ
  • dataпјҡжҹҘиҜўе·ІжңүеҜ№иұЎ

Variable / Output вҖ” иҫ“е…ҘдёҺиҫ“еҮә

  • variableпјҡз»ҷй…ҚзҪ®дј еҸӮ
  • outputпјҡжҠҠе…ій”®з»“жһңеҜјеҮәз»ҷдәәзңӢпјҢжҲ–з»ҷе…¶д»–жЁЎеқ—дҪҝз”Ё

Module вҖ” еҸҜеӨҚз”Ёй…ҚзҪ®еҚ•е…ғ

еҪ“дёҖз»„иө„жәҗз»ҸеёёдёҖиө·еҮәзҺ°пјҢдҫӢеҰӮвҖңVPC + еӯҗзҪ‘ + и·Ҝз”ұиЎЁ + NATвҖқпјҢе°ұеә”иҜҘжҠҪжҲҗ moduleгҖӮ

дҪ еҸҜд»ҘжҠҠ module зҗҶи§ЈжҲҗвҖңеҹәзЎҖи®ҫж–ҪеҮҪж•°вҖқпјҡ

  • иҫ“е…Ҙпјҡvariables
  • еҶ…йғЁпјҡеӨҡдёӘ resources
  • иҫ“еҮәпјҡoutputs

Terraform зҡ„е·ҘзЁӢеҢ–иғҪеҠӣпјҢеҫҲеӨ§зЁӢеәҰдёҠе°ұе»әз«ӢеңЁ module еӨҚз”ЁдёҠгҖӮ

4. StateпјҡTerraform дёәд»Җд№ҲйңҖиҰҒзҠ¶жҖҒж–Үд»¶пјҹ

S: Terraform дёҚжҳҜе·Із»Ҹжңү .tf й…ҚзҪ®дәҶеҗ—пјҹ C: й…ҚзҪ®еҸӘжҸҸиҝ°вҖңжҲ‘жғіиҰҒд»Җд№ҲвҖқпјҢдҪҶдёҚзҹҘйҒ“вҖңзҺ°еңЁе·Із»Ҹжңүд»Җд№ҲвҖқгҖҒиө„жәҗ ID жҳҜеӨҡе°‘гҖҒе“ӘдәӣеҜ№иұЎеҪ’жҲ‘з®ЎзҗҶ Q: Terraform йқ д»Җд№ҲиҝҪиёӘиө„жәҗпјҹ A: state ж–Үд»¶пјҢй»ҳи®ӨжҳҜ terraform.tfstate

state зҡ„дҪңз”Ёпјҡ

  • и®°еҪ• Terraform з®ЎзҗҶиҝҮзҡ„иө„жәҗеҸҠе…¶еұһжҖ§
  • и®°еҪ•иө„жәҗзңҹе®һ IDпјҲдҫӢеҰӮе®һдҫӢ IDгҖҒVPC IDпјү
  • дҪңдёә plan зҡ„жҜ”иҫғеҹәеҮҶ
  • её®еҠ© Terraform е»әз«Ӣдҫқиө–еӣҫе’ҢеҸҳжӣҙи·Ҝеҫ„

еҰӮжһңжІЎжңү stateпјҢTerraform еҫҲйҡҫеҲӨж–ӯпјҡ

  • дёҖдёӘиө„жәҗжҳҜвҖңе·Із»ҸеӯҳеңЁпјҢеә”дҝ®ж”№вҖқ
  • иҝҳжҳҜвҖңж №жң¬дёҚеӯҳеңЁпјҢеә”ж–°е»әвҖқ

жң¬ең° state дёҺиҝңз«Ҝ state

е…Ҙй—Ёж—¶еёёи§ҒжҳҜжң¬ең° terraform.tfstateпјҢдҪҶеӣўйҳҹеҚҸдҪңйҖҡеёёиҰҒж”ҫеҲ°иҝңз«Ҝ backendпјҢдҫӢеҰӮпјҡ

  • S3 + DynamoDB Lock
  • йҳҝйҮҢдә‘ OSS
  • Terraform Cloud

иҝңз«Ҝ state зҡ„д»·еҖјпјҡ

  • еӣўйҳҹе…ұдә«
  • йҒҝе…ҚеӨҡдәәеҗ„иҮӘжҢҒжңүдёҚеҗҢзҠ¶жҖҒ
  • жҸҗдҫӣй”ҒпјҢйҳІжӯўе№¶еҸ‘ apply

7. Terraform зҡ„дҫқиө–е…ізі»жҳҜжҖҺд№ҲжҺЁеҮәжқҘзҡ„пјҹ

  • еҶҚеҲӣе»ә aws_subnet.app
