S 排查 DNS 解析问题时用。适合确认某个域名当前解析到什么 IP、权威记录是否正确、不同 DNS 服务器返回是否一致、递归链路哪里出了问题。

C 常见冲突：

  • 浏览器访问失败，但不确定是网络问题还是 DNS 问题
  • 域名刚改解析，怀疑本地缓存、公共 DNS、权威 DNS 返回不一致
  • 邮件、子域名、CDN、负载均衡配置异常，想确认 MX / CNAME / NS 等记录
  • 某台机器能解析，另一台不能解析，想对比指定 DNS 服务器的返回结果

Q 排查时常问：

  • 这个域名现在解析到了什么记录
  • 是哪个 DNS 服务器给出的答案
  • 权威 DNS 和递归 DNS 返回是否一致
  • 结果是不是被简化了，TTL 是多少，是否还在传播中
  • 某个 IP 反查回来的域名是什么
