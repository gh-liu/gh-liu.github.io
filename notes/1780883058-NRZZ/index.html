Hooks 一览（man githooks, Git 2.43）

钩子放在 $GIT_DIR/hooks/（或 core.hooksPath），需有可执行位才生效。

提交相关（client-side）

  • pre-commit：git commit 前触发，无参数；非零退出会中止提交。默认钩子检查行尾空白/非 ASCII 文件名。可用 --no-verify 跳过。
  • prepare-commit-msg：准备好默认提交信息后、打开编辑器前触发；用于改写信息文件，--no-verify 不跳过。
  • commit-msg：拿到提交信息文件后触发，参数为信息文件名；可校验/改写信息，非零退出中止。
  • post-commit：提交完成后触发，仅用于通知，不能影响结果。
  • pre-merge-commit：git merge 成功合并后、生成提交信息前触发；默认转调 pre-commit。
  • post-merge：git merge/git pull 完成后触发，参数为是否 squash；冲突失败时不执行。
  • post-rewrite：git commit --amend 或 git rebase 重写提交后触发，stdin 给出 old→new 映射。
  • pre-rebase：git rebase 前触发，可阻止分支被 rebase。
  • post-checkout：git checkout/switch/clone/worktree add 更新工作树后触发，参数为旧 HEAD、新 HEAD、是否分支切换。
  • pre-auto-gc：git gc --auto 前触发，非零退出中止自动 gc。
  • post-index-change：索引写入时触发，参数指示工作树是否更新、skip-worktree 位是否变化。
  • reference-transaction：任何引用更新事务触发，参数为 prepared/committed/aborted，stdin 给出 old/new/ref。

git am 打补丁相关

  • applypatch-msg：git am 时触发，参数为提交信息文件名；可校验/改写，非零退出中止。
  • pre-applypatch：补丁已应用、提交前触发，无参数；非零退出则不提交。
  • post-applypatch：补丁应用并提交后触发，仅用于通知。

推送 / 服务端相关

  • pre-push：git push 前在本地触发，参数为远程名和 URL，stdin 给出待推送 ref；非零退出中止推送。
  • pre-receive：服务端收到推送、更新 ref 前触发一次，stdin 给出所有 old/new/ref；非零退出拒绝全部更新。
  • update：服务端对每个待更新 ref 触发一次，参数为 ref 名、old、new；可实现 fast-forward only 等策略。
  • proc-receive：服务端按 receive.procReceiveRefs 匹配的 ref 改由该钩子用 pkt-line 协议处理更新。
  • post-receive：服务端所有 ref 更新后触发一次，stdin 同 pre-receive；用于通知，不影响结果。
  • post-update：服务端 ref 更新后触发，参数为更新的 ref 名；默认运行 update-server-info（HTTP dumb 传输）。
  • push-to-checkout：推送到已检出分支且 receive.denyCurrentBranch=updateInstead 时触发，自定义工作树/索引更新行为。
