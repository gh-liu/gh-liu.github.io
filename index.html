1 · Note

From smaller composable ideas to complex ideas.

1.1 · Workflow

目的：知识可累积、可组合、可产出

捕捉想法（随时）-> 浅加工（每天）-> 深加工（每周）-> 输出（博客）

  1. 捕捉想法（快速、不判断分类）：telegram bot
  2. 浅加工：daily note: daily/YYYY-MM-DD.md
  3. 深加工：按主题进行分类总结（长青笔记），笔记之间进行链接（密集）
  4. 输出博客
  5. 个人展示：NOW 页面，展示目前在干什么？对什么感兴趣？当下的思考（更广的时间跨度）？
  • 长青笔记: 是概念导向、原子化、密集链接、持续演化的知识单位；定义了笔记应该是怎么样的
  • 卡片系统: 通过唯一ID和内部链接构建知识网络，让结构自下而上涌现；定义了笔记之间的关系
  • 记忆媒介：笔记既是思考的载体，也是记忆的训练场；定义了如何记住笔记

临时笔记-长青笔记-博客关系：

  • 临时笔记(daily notes)是原始素材
  • 长青笔记(evergreen notes)是提炼加工后的成果
  • 博客是 evergreen notes 的公开版本

1.2 · Pipeline

┌────────────────────────────────────────┐
│ Layer 1: 捕获（Capture）               │
│ 入口：快速记录工具 → 临时 inbox        │
│ 原则：零摩擦，不判断、不分类           │
└────────────────────────────────────────┘
                  ↓ 自动定期同步
┌────────────────────────────────────────┐
│ Layer 2: 浅加工（Daily Processing）    │
│ 位置：每日笔记                         │
│ 内容：自动同步 + 手动补充              │
│ 策略：定期归档                         │
└────────────────────────────────────────┘
                  ↓ 定期手动深加工
┌────────────────────────────────────────┐
│ Layer 3: 深加工（Curation）            │
│ 操作：提炼为 evergreen notes           │
│ 原则：概念导向、原子化、至少2个出链    │
└────────────────────────────────────────┘
                  ↓ 按需输出
┌────────────────────────────────────────┐
│ Layer 4: 输出（Publishing）            │
│ 操作：组合笔记 → 发布文章              │
│ 闭环：标注源链接，形成追溯链路         │
└────────────────────────────────────────┘

1.3 · Setup

notes/
├── daily/          # 每日笔记（临时，定期归档）
├── evergreen*.md   # 常青笔记（永久积累，放根目录）
└── references/     # 参考资料索引
  1. 笔记中首次提到某个概念时，就加上链接，即使链接是未完成的（尚无对应笔记）
  2. 引用的（文章、电影等）有一条在References文件夹中的笔记，包含其相关属性、自己评分、自己的想法
  3. 每日随机笔记用(YYYY-MM-DD_HHmm_TITLE)，每几天进行回顾提炼，每月汇总，每年回顾；每几个月随机回访笔记
  4. 尽可能从模版创建笔记，每个模版预先嵌入一些信息（用于高效检索、关联笔记）
  5. tgbot->issues->daily（自动化）

指标?

维度指标目标
捕获每周捕获数量持续稳定
加工每周新建 evergreen notes有节奏
链接每篇笔记的平均出链数≥2
产出每月输出文章数有节奏
复用旧笔记被引用次数持续增长
