1 · 认知 Loop#

积累 -> 消化思考 -> 表达观点 -> 讨论检验 -> 修补漏洞 -> 下一轮

这套笔记系统的目标，不是囤积信息，而是把对内的思考转化为对外的观点。

也就是说，要把模糊的感受、零散的见闻、未经整理的想法，逐步变成可表达、可验证、可反驳的内容。笔记不是资料仓库，而是思考的载体；更进一步说，笔记即你的观点。

高质量的讨论可以视为一次压力测试。它会暴露思考中的盲区、论证中的漏洞，以及表达上的含混。盲区出现后，再进入下一轮思考、表达与检验，认知 Loop 也就继续向前。

在写作层面：

  • 用 SCQA 框架 写序言，说明背景、冲突与核心问题
  • 用 MECE 分析法 做分析论证，尽量让结构完整且不重叠

1.1 · 核心原则

  • 笔记服务于思考，不服务于收藏
  • 每条笔记尽量表达一个清晰的观点或概念
  • 链接不是装饰，而是知识网络的结构
  • 输出不是终点，讨论和修补也属于笔记系统的一部分

1.2 · Workflow#

From smaller composable ideas to complex ideas.

目的：知识可累积、可组合、可产出

捕捉想法（随时）-> 浅加工（每天）-> 深加工（每周）-> 输出（博客）

1.2.1 · 1. 捕捉想法#

原则是快速记录、低摩擦、不急着判断和分类。

当前入口：

  • Telegram bot

1.2.2 · 2. 浅加工#

把原始素材放进每日笔记中，先做轻量整理，不要求一次成型。

常见方式：

  • 写入 daily/YYYY-MM-DD.md
  • 或直接进入长青笔记，但标记 draft=true
  • 必要时可借助 AI 做初步摘要、提炼问题或整理条目

1.2.3 · 3. 深加工#

定期把 daily notes 中反复出现、值得保留的内容提炼成 长青笔记。

深加工时重点做三件事：

  • 提炼概念，而不是重复素材
  • 建立链接，让笔记进入知识网络
  • 补足论证，让内容从“记录”变成“观点”

1.2.4 · 4. 输出#

当一组相关的 evergreen notes 已经足够成熟，就可以把它们组合成博客文章或其他公开表达。

输出的价值不只是发布，更重要的是：

  • 借公开表达倒逼结构化思考
  • 借讨论反馈发现盲区
  • 借源链接保留可追溯链路

1.3 · 三层内容的关系

  • 临时笔记（daily notes）是原始素材
  • 长青笔记（evergreen notes）是提炼加工后的成果
  • 博客是 evergreen notes 的公开版本

这三者不是彼此替代，而是同一条生产链上的不同阶段。

1.4 · 相关概念

  • 长青笔记：概念导向、原子化、密集链接、持续演化的知识单位，定义“笔记应该是什么”
  • 卡片系统：通过唯一 ID 和内部链接构建知识网络，让结构自下而上涌现，定义“笔记之间如何连接”
  • 记忆媒介：笔记既是思考的载体，也是记忆的训练场，定义“为什么要反复回访与重用”

1.5 · Pipeline#

┌────────────────────────────────────────┐
│ Layer 1: 捕获（Capture）               │
│ 入口：快速记录工具 -> 临时 inbox       │
│ 原则：零摩擦，不判断、不分类           │
└────────────────────────────────────────┘
                  ↓ 自动定期同步
┌────────────────────────────────────────┐
│ Layer 2: 浅加工（Daily Processing）    │
│ 位置：每日笔记                         │
│ 内容：自动同步 + 手动补充              │
│ 策略：定期归档                         │
└────────────────────────────────────────┘
                  ↓ 定期手动深加工
┌────────────────────────────────────────┐
│ Layer 3: 深加工（Curation）            │
│ 操作：提炼为 evergreen notes           │
│ 原则：概念导向、原子化、至少 2 个出链  │
└────────────────────────────────────────┘
                  ↓ 按需输出
┌────────────────────────────────────────┐
│ Layer 4: 输出（Publishing）            │
│ 操作：组合笔记 -> 发布文章             │
│ 闭环：标注源链接，形成追溯链路         │
└────────────────────────────────────────┘

1.6 · 目录结构

notes/
├── daily/          # 每日笔记：临时素材，定期归档
├── evergreen*.md   # 长青笔记：永久积累，放在 notes/ 根目录
└── references/     # 参考资料索引

1.7 · 使用规则

  1. 首次提到某个概念时就加上内部链接，即使目标笔记还不存在。
  2. 每个被引用的对象（文章、电影、播客、人物等）尽量在 references/ 中有对应笔记，记录其基本信息、评分和自己的想法。
  3. 临时笔记使用类似 YYYY-MM-DD_HHmm_TITLE 的命名方式，方便排序和回顾。
  4. daily notes 需要定期回看，做提炼、归档和迁移，而不是长期堆积。
  5. 尽可能从模版创建笔记，让元数据、关联字段和常用结构保持一致。
  6. evergreen notes 尽量满足“概念明确、论点单一、至少 2 个出链”。
  7. 自动化链路保持简单明确：tgbot -> issues -> daily。

1.8 · 建议节奏

  • 随时捕获想法
  • 每天整理 daily notes
  • 每周做一次深加工
  • 每月做一次主题汇总或输出
  • 每隔几个月随机回访旧笔记，检查是否可复用、更新或重写

1.9 · 衡量指标

维度指标目标
捕获每周捕获数量持续稳定
加工每周新建 evergreen notes有节奏
链接每篇笔记的平均出链数>= 2
产出每月输出文章数有节奏
复用旧笔记被引用次数持续增长

1.10 · 其他想法

“Evergreen notes turn ideas into objects that you can manipulate, combine, recombine.” — Steph Ango

“Creativity is defined by the uniqueness and appeal of the combination of elements.” — Steph Ango

“Better note-taking misses the point; what matters is better thinking.” — Andy Matuschak

可以补充一个 NOW 页面，用来展示：

  • 目前在做什么
  • 最近对什么感兴趣
  • 当下正在形成的思考

它不是日记，而是一个时间跨度更宽的个人状态页。
